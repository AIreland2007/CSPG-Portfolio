# Airgead Banking Investment Calculator

## Overview

The Airgead Banking Investment Calculator is a C++ console application developed to help users understand how compound interest and recurring monthly deposits affect long-term investment growth. The application accepts an initial investment amount, monthly deposit, annual interest rate, and investment period, then generates yearly reports comparing account growth with and without additional monthly deposits. This allows users to clearly visualize the impact of consistent investing over time.

## Reflection

This project strengthened my understanding of object-oriented programming and reinforced the importance of designing software that is organized, maintainable, and user-friendly. I structured the program using separate header, implementation, and main source files, making the code easier to read, test, and expand. I also incorporated input validation to improve reliability and ensure users entered valid financial data before calculations were performed.

One of the more challenging aspects of this project was implementing the compound interest calculations while accurately accounting for monthly deposits throughout each investment period. I approached this challenge by breaking the problem into smaller steps, carefully testing different investment scenarios, and refining the calculation logic until the generated reports produced consistent and accurate results.

If I were to continue developing this application, I would enhance the user experience by allowing different compounding frequencies, exporting reports to a file, and providing a graphical user interface. These improvements would increase the program's flexibility, usability, and scalability while demonstrating more advanced software engineering practices.

This project also reinforced several programming concepts that are directly transferable to future coursework and professional software development, including object-oriented design, modular programming, input validation, loops, functions, and financial calculations. Most importantly, it emphasized writing code that is readable, maintainable, and adaptable by separating responsibilities into logical components, using meaningful variable names, and minimizing repetitive code.
